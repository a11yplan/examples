WCAG 3.2.4: Consistent Identification

Success Criterion 3.2.4 Consistent Identification (Level AA)

Requirement: Components that have the same functionality within a set of Web pages are identified consistently.

Purpose: Helps users learn the interface and predict where to find specific functionality. Particularly beneficial for users with cognitive limitations, screen reader users who rely on consistent labels, and users who benefit from predictable interfaces.

Scope: Applies to buttons, links, icons, form fields, and any interactive component that appears on multiple pages with the same function. Labels, alt text, and aria-labels must be consistent when functionality is the same.

Note: This criterion focuses on consistency of identification, not position. Components with different functions may use different labels, even if visually similar.

Expected Test Results

  • Total Test Cases: 10 (5 violations + 5 passes)
  • Expected Violations: 5 (TC1.1 - TC1.5) - Same functionality identified inconsistently
  • Expected Passes: 5 (TC2.1 - TC2.5) - Same functionality identified consistently
  • Test Categories:
    • Button labels and icons
    • Search functionality
    • Navigation controls
    • Action buttons (save, delete, download)

Important Notes

Consistent Identification Applies To:

  • Button labels: Text on buttons with same function
  • Link text: Anchor text for similar destinations
  • Icons: Visual indicators for same actions
  • Alt text: Image alternatives for functional images
  • ARIA labels: Accessible names for controls
  • Form field labels: Labels for similar input fields

Different Functions May Use Different Labels:

If two buttons have different functions (e.g., "Save Draft" vs "Save and Publish"), they should have different labels to clarify the different outcomes.

Testing: Review multiple pages and verify that components with identical functionality use identical labels, icons, and accessible names.
