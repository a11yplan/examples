WCAG 2.4.7 Focus Visible - Test Examples

This page demonstrates both failing and passing examples of WCAG 2.4.7 Focus Visible. Use keyboard navigation (Tab key) to see the differences.

❌ FAILING Examples (WCAG 2.4.7 Violation)

Problem: These elements have their focus indicators removed with outline: none and provide no alternative visible indicator. Keyboard users cannot see which element has focus.
Link without Focus Indicator
/* WCAG 2.4.7 FAILURE */ .btn-no-focus:focus { outline: none; /* Removes focus indicator */ } .link-no-focus:focus { outline: none; /* No visible focus */ text-decoration: none; /* Removes underline too */ } .input-no-focus:focus { outline: none; /* No alternative provided */ }

✅ PASSING Examples (WCAG 2.4.7 Compliant)

Solution: These elements provide clear, high-contrast visible focus indicators. Keyboard users can easily see which element currently has focus.
Link with Clear Focus
/* WCAG 2.4.7 SUCCESS */ .btn-visible-focus:focus { outline: 3px solid #f39c12; outline-offset: 3px; box-shadow: 0 0 0 5px rgba(243, 156, 18, 0.3); } .link-visible-focus:focus { outline: 3px solid #e67e22; outline-offset: 2px; background-color: #fff3e0; } .input-visible-focus:focus { outline: 3px solid #3498db; outline-offset: 2px; border-color: #2980b9; }

Testing Instructions:

  • Keyboard Test: Press Tab to navigate through all interactive elements
  • Expected Behavior:
    • Failing examples (red section): No visible indication of focus
    • Passing examples (green section): Clear orange/blue outline when focused
  • WCAG 2.4.7 Requirement: Any keyboard operable user interface must have a mode of operation where the keyboard focus indicator is visible
  • Common Failures:
    • Using outline: none without providing an alternative
    • Focus indicator has insufficient contrast with background
    • Focus indicator is too small or subtle to notice
  • Expected Scanner Results: Your accessibility scanner should flag the elements in the red section as WCAG 2.4.7 violations
