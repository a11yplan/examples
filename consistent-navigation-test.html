WCAG 3.2.3: Consistent Navigation

Success Criterion 3.2.3 Consistent Navigation (Level AA)

Requirement: Navigational mechanisms that are repeated on multiple Web pages within a set of Web pages occur in the same relative order each time they are repeated, unless a change is initiated by the user.

Purpose: Helps users with cognitive limitations, low vision, and intellectual disabilities by allowing them to develop a mental model of where navigation elements appear. Consistent positioning reduces cognitive load and makes sites easier to navigate.

Scope: Applies to navigation menus, search boxes, and other repeated navigation mechanisms. Does not prevent adding/removing items, only requires maintaining relative order of repeated items.

Expected Test Results

  • Total Test Cases: 10 (5 violations + 5 passes)
  • Expected Violations: 5 (TC1.1 - TC1.5) - Inconsistent navigation order across pages
  • Expected Passes: 5 (TC2.1 - TC2.5) - Consistent navigation order maintained
  • Note: Each test case simulates multiple pages to demonstrate consistency (or lack thereof)

Test Case 1: Navigation Menu Reordering

❌ FAIL: Navigation Order Changes

Page: Home
Home About Services Contact
Order: Home, About, Services, Contact
Page: Services
Home Services About Contact
Order changed: Home, Services, About, Contact
Violation: Navigation items change order between pages. "Services" and "About" swap positions, forcing users to relearn navigation on each page.

✅ PASS: Navigation Order Consistent

Page: Home
Home About Services Contact
Order: Home, About, Services, Contact
Page: Services
Home About Services Contact
Same order maintained: Home, About, Services, Contact
Pass: Navigation items maintain the same relative order on all pages. Users can predict where each link will be located.
<!-- PASS: Consistent order --> <!-- All pages use same order --> <nav>Home | About | Services | Contact</nav>

Test Case 2: Section-Based Reordering

❌ FAIL: Priority-Based Reordering

Page: Products Section
Home Products About Blog Contact
Products section: Products moved to position 2
Page: Blog Section
Home Blog About Products Contact
Blog section: Blog moved to position 2, Products to position 4
Violation: Navigation items rearranged to prioritize the current section. While this seems helpful, it confuses users who rely on consistent positioning.
<!-- FAIL: Context-based reordering --> <!-- Products page --> <nav>Home | Products | About | Blog | Contact</nav> <!-- Blog page --> <nav>Home | Blog | About | Products | Contact</nav>

✅ PASS: Fixed Order with Visual Indicator

Page: Products Section
Home About Products Blog Contact
Products section: Order maintained, current page highlighted
Page: Blog Section
Home About Products Blog Contact
Blog section: Same order, visual highlighting shows current page
Pass: Navigation order stays the same. Visual styling (highlighting) indicates current page without changing position.
<!-- PASS: Fixed order with active state --> <!-- All pages use same order --> <nav> <a href="#">Home</a> <a href="#">About</a> <a href="#" class="active">Products</a> <a href="#">Blog</a> <a href="#">Contact</a> </nav>

Important Notes

What IS allowed:

  • Adding new items to navigation (at beginning or end)
  • Removing items from navigation on some pages
  • User-initiated changes (e.g., personalized navigation order)
  • Visual styling changes (highlighting current page)

What is NOT allowed:

  • Changing the relative order of repeated navigation items
  • Moving navigation elements to different positions
  • Context-based reordering without user control

Testing: Navigate through multiple pages of a site and verify that navigation elements appear in the same relative order on each page.
