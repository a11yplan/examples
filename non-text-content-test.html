WCAG 1.1.1 Requirements:

  • All non-text content (images, SVG, etc.) must have text alternatives
  • Decorative images should have empty alt (alt="")
  • Informative images need descriptive alt text
  • This criterion focuses on non-interactive elements
  • Interactive elements (buttons, inputs) fall under other criteria (e.g., 4.1.2)
  • Level: A

Test Structure:

Each test case shows the SAME content side-by-side:

  • Left (Red/FAIL): Incorrect implementation
  • Right (Green/PASS): Correct implementation

Poor Alt Text Quality Patterns (TC1.3-1.6):

  • Filenames: Alt text like "1214.jpg" or "IMG_5432.png"
  • Special Characters/Encoding: Alt text like ">", "П", or "√É¬©"
  • URLs/Paths: Alt text like "/assets/images/hero.jpg"
  • Single Generic Words: Alt text like "Placeholder", "Foto", "Image"

AI Testing Capabilities:

  • Vision models can detect informative charts/diagrams (TC1.1, TC1.2)
  • Can verify alt text accuracy by comparing description to visual content (TC2.1, TC2.2)
  • Can identify poor quality patterns (filenames, URLs, generic words)
  • Can detect redundant labeling when icons have labels within labeled buttons (TC1.7)

Special Case - Decorative Icons in Buttons (TC1.7):

  • When a button has visible text, icon SVGs inside should be hidden with aria-hidden="true"
  • Without aria-hidden, screen readers announce both icon label AND button text
  • Example FAIL: "Settings icon, Settings, button" (redundant)
  • Example PASS: "Settings, button" (clear and concise)
