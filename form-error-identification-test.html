WCAG Requirements:

  • 3.3.1 Error Identification (Level A): If an input error is automatically detected, the item in error is identified and described in text
  • 3.3.3 Error Suggestion (Level AA): If an input error is detected and suggestions for correction are known, the suggestions are provided to the user

Test Structure:

Each test case shows the SAME form field with validation errors side-by-side:

  • Left (Red/FAIL): Poor visual indication, generic/cryptic error messages, no programmatic error state
  • Right (Green/PASS): Clear visual indication, specific error messages, helpful suggestions, proper ARIA attributes

Common Violations:

  • Poor Visual Indication: Subtle color changes, no icons, easily missed errors
  • Generic Messages: "Error", "Invalid", "X", "!" without context
  • No Suggestions: Doesn't explain how to fix the error
  • No ARIA: Missing aria-invalid, aria-describedby, aria-required
  • No Programmatic Link: Error message not associated with field

Passing Requirements:

  • Clear Visual Design: Strong border colors, background changes, error icons
  • Specific Messages: Describe exactly what's wrong (e.g., "Email format is invalid")
  • Helpful Suggestions: Explain how to fix (e.g., "Enter email with @ and domain")
  • aria-invalid="true": On the input field or container (for radio groups)
  • aria-describedby: Links input field to error message element
  • role="alert": On error message for immediate screen reader announcement
  • aria-required="true": On required fields

AI Testing Capabilities:

  • Can detect presence/absence of aria-invalid attribute
  • Can verify aria-describedby links to error message element
  • Can analyze error message text quality (generic vs. specific)
  • Can detect presence of correction suggestions in error messages
  • Can identify role="alert" on error messages
  • Can evaluate visual error indicators (border color, background, icons)
  • Can verify error messages are programmatically associated with form fields

Benefits for Users:

  • Screen Reader Users: Error state announced, error description read, suggestion provided
  • Keyboard Users: Can navigate to error message from field
  • Low Vision Users: Strong visual indicators make errors obvious
  • Cognitive Disabilities: Clear explanations and suggestions reduce confusion
  • All Users: Faster form completion with specific guidance
