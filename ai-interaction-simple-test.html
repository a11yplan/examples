Test Purpose

This page contains 6 simple interaction tests designed to validate AI browser capabilities. Each test requires the AI to perform a basic action and verify the expected outcome.

Target Completion Time: ~30 seconds

Validation Method: AI must verify DOM state, element properties, and visible text changes

Test Case 1: Button Click Pending
AI Instruction: Click the button labeled "Click Me!" below.
Expected Outcome: The button text should change from "Click Me!" to "Button Clicked!" and the status badge should change to "Completed".

Validation Checkpoints:

  • Button text equals "Button Clicked!"
  • Button's data-clicked attribute is "true"
  • Test card has class "completed"
  • Status badge shows "Completed"
Test Case 2: Text Input Pending
AI Instruction: Type "AI Test Input" into the text input field below.
Expected Outcome: The input field should contain "AI Test Input" and a confirmation message should appear below showing the typed text.

Validation Checkpoints:

  • Input value equals "AI Test Input"
  • Result div contains text: "You typed: AI Test Input"
  • Test card has class "completed"
  • Status badge shows "Completed"
Test Case 3: Checkbox Toggle Pending
AI Instruction: Click the checkbox to enable the feature.
Expected Outcome: The checkbox should be checked and a confirmation message should appear stating "Feature enabled!".

Validation Checkpoints:

  • Checkbox checked property is true
  • Result div contains text: "Feature enabled!"
  • Result div has class "success-message"
  • Test card has class "completed"
Test Case 4: Dropdown Selection Pending
AI Instruction: Select "Option 2" from the dropdown menu.
Expected Outcome: The dropdown should show "Option 2" as selected and a confirmation message should display "You selected: Option 2".

Validation Checkpoints:
